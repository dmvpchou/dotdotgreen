# 點點綠市集公開資料彙整

> 最後查核：2026-07-15（Asia/Taipei）
> 用途：官網內容、品牌 Content Collection 與未來出攤資料介面的單一查核入口。
> 原則：營業資訊以主辦方現行頁面／粉專為準；媒體文章只用於歷史、品牌與品項背景，不把舊週表當成現在班表。

## 接手結論

此專案可以直接在 Codex 繼續開發。Git repo、Astro 專案、Content Collection、六頁 UI、設計 tokens 與原生互動程式都已存在，不需要重新 scaffold。

接手時辨識到的主要風險不是框架，而是資料可信度：

1. 原程式的市集時間、週間營業日與三站捷運編號和現行公開資料不一致。
2. 品牌介紹、固定出攤日與站點分配原本多為 Design UI Kit 的示範稿。
3. 每週出攤會隨產季、天候與請假變動，不能只靠 `attendDays` 做永久循環月曆。
4. 品牌社群連結多為 `#` 佔位，不可當成正式外連。

本輪已先校正高可信度的營業資訊，並把品牌內容改成可追溯公開來源的版本。即時班表在取得主辦方可用資料前，不在官網自行推算。2026-07-15 再以瀏覽器查核 Facebook 公開頁面，確認北投站已於 2026-07-01 初登場，週間模式現為四站。

## 資料來源優先順序

| 優先 | 來源 | 適合用途 | 限制 |
|---|---|---|---|
| 1 | 點點綠／協會現行頁面 | 營業時間、站點、主辦／協辦、聯絡方式 | 每週品牌仍導向 Facebook，沒有結構化班表 |
| 2 | 點點綠 Facebook 粉專 | 當週品牌、臨時異動、最新活動 | 內容不一定能穩定由搜尋引擎或程式讀取 |
| 3 | 臺北市政府／臺北捷運 | 合作背景、活動事實、站號 | 多為特定日期活動或歷史新聞稿 |
| 4 | 綠媒體／協會文章 | 品牌名稱、品項背景、品牌故事 | 文章內的週表是歷史快照，不代表目前仍出攤 |
| 5 | 攤商品牌自己的社群 | 品項、品牌介紹、個別出攤公告 | 需逐品牌確認官方帳號與授權 |

## 待主辦方補充的資料

以下內容不應只靠網路推測：

- 目前有效的完整品牌名冊、下架狀態與品牌排序。
- 每個品牌的官方 Facebook／Instagram／LINE／網站網址。
- 品牌自介、品牌照與照片使用授權。
- 當週「日期＋站點＋品牌」班表及臨時異動方式。
- 各站固定營業日（若確實固定）與北投站試營運期限／常態化條件。
- 雙連 1 號與 2 號出口在平日、假日與免費場地合作中的分工。
- 目前有效的加入／舊夥伴／場地表單及各自 owner。
- 假日市集時段是否有季節性調整規則。
- `weekday` 莓紫分類色是否定案。

## 出攤資料模型建議

現有品牌 frontmatter 的 `attendDays: ["二", "四"]` 只能表達永久循環星期，無法處理產季、請假、換站與單週活動。若恢復週曆／月曆，建議另建日期型資料：

```yaml
date: 2026-07-17
market: weekday
station: shilin
brands:
  - brand-slug
status: confirmed
sourceUrl: https://...
updatedAt: 2026-07-14T18:00:00+08:00
```

品牌檔只管理相對穩定的品牌資料；出攤檔管理每日事件。前台只顯示 `confirmed` 資料，沒有資料時直接導向官方粉專。
